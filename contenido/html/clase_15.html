Clase 15: Árboles Balanceados

Garantizar O(log n) en todos los casos

El problema del BST no balanceado

Un BST con inserción ordenada degenera en una lista. Los árboles balanceados impiden esto manteniendo la altura controlada: O(log n) garantizado.

Factor de balance (AVL)

El factor de balance de un nodo es: altura(subárbol derecho) − altura(subárbol izquierdo). En un árbol AVL, este valor debe ser -1, 0 o +1 para todo nodo.

struct NodoAVL {
    int  valor, altura;
    NodoAVL *izq, *der;
};

int altura(NodoAVL* n) { return n ? n->altura : 0; }
int balance(NodoAVL* n) { return n ? altura(n->der)-altura(n->izq) : 0; }

Rotaciones AVL

// Rotación simple derecha (caso LL)
NodoAVL* rotDerecha(NodoAVL* y) {
    NodoAVL* x  = y->izq;
    NodoAVL* T2 = x->der;
    x->der = y;  y->izq = T2;
    y->altura = 1 + max(altura(y->izq), altura(y->der));
    x->altura = 1 + max(altura(x->izq), altura(x->der));
    return x;  // nueva raíz
}

Árboles Red-Black

Alternativa al AVL, usada en la STL (map, set). Sus 5 propiedades garantizan que la altura sea siempre O(log n):

  1. Todo nodo es rojo o negro.
  2. La raíz es negra.
  3. Las hojas (NULL) son negras.
  4. Un nodo rojo sólo puede tener hijos negros.
  5. Todo camino desde un nodo hasta sus hojas tiene el mismo número de nodos negros (black-height).
