Proceso de revisión final

Revisión de código (Code Review)

Lee tu propio código como si fuera de otra persona. Pregúntate:

  • ¿Es claro lo que hace cada función con solo leer su nombre y comentarios?
  • ¿Hay código duplicado que pueda refactorizarse?
  • ¿Cada función hace una sola cosa (principio de responsabilidad única)?

Pruebas de estrés

// Probar con volúmenes grandes para verificar eficiencia
#include <chrono>

auto inicio = chrono::high_resolution_clock::now();

// Insertar 100 000 elementos en tu estructura
for (int i=0; i<100000; i++) estructura.insertar(rand());

auto fin = chrono::high_resolution_clock::now();
auto duracion = chrono::duration_cast<chrono::milliseconds>(fin-inicio);
cout << "Tiempo: " << duracion.count() << " ms\n";
// Si excede 1-2 segundos → revisar complejidad

Refactorización común: separar interfaz de implementación

// Archivo: bst.h — solo declaraciones
class BST {
public:
    void   insertar(int v);
    bool   buscar(int v) const;
    void   eliminar(int v);
    void   imprimirInorden() const;
    int    getAltura() const;
private:
    Nodo*  raiz = nullptr;
    Nodo*  insertar(Nodo* r, int v);
    // ...
};

// Archivo: bst.cpp — implementaciones
#include "bst.h"
void BST::insertar(int v) { raiz = insertar(raiz, v); }
// ...

Preparar la presentación

Una buena presentación del proyecto debe mostrar:

  1. El problema: ¿qué resuelve el proyecto? (30 segundos)
  2. La solución: ¿qué estructuras se usaron y por qué? (2 minutos)
  3. Demo en vivo: ejecutar con casos de prueba representativos. (2 minutos)
  4. Análisis de complejidad: tabla de operaciones y sus costos. (1 minuto)
  5. Conclusiones: ¿qué cambiarías con más tiempo? (30 segundos)
