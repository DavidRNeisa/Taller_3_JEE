Clase 7: Multilistas y Composición de Estructuras

Modelar relaciones complejas con estructuras interconectadas

¿Qué es una multilista?

Una multilista es una estructura donde los nodos participan simultáneamente en múltiples listas. Cada nodo tiene más de un puntero "siguiente", uno por cada lista a la que pertenece.

Caso real: Un estudiante pertenece a la vez a la lista de su carrera, a la de cada materia y a la de su dormitorio. Un único nodo participa en varias listas sin duplicar datos.

Estructura de nodo multi-enlazado

struct Estudiante {
    string nombre;
    int    codigo;
    Estudiante* sigCarrera;    // enlace en la lista de su carrera
    Estudiante* sigMateria;    // enlace en la lista de la materia
    Estudiante* sigDormitorio; // enlace en la lista del dormitorio
};

Actualizar el nombre del estudiante lo actualiza en todas las vistas simultáneamente, sin inconsistencias.

Matrices dispersas con multilistas

Una matriz dispersa tiene la mayoría de sus elementos en cero. Guardar todos los n² valores desperdiciaría memoria. La multilista solo almacena los no-cero, con enlaces por fila y por columna:

struct NodoMatriz {
    int    fila, columna;
    double valor;
    NodoMatriz* sigFila;    // siguiente en la misma fila
    NodoMatriz* sigColumna; // siguiente en la misma columna
};
// Matriz 1000×1000 con 1000 elementos ≠ 0:
// Sin multilista: 1 000 000 entradas
// Con multilista: 1 000 nodos — ahorro de 99.9%

Composición de estructuras

La composición construye estructuras complejas combinando simples (principio "has-a"):

struct Punto { double x, y; };

struct Rectangulo {
    Punto esquinaSupIzq;
    Punto esquinaInfDer;
};

struct Grafo {
    vector<string>       vertices;
    vector<vector<int>>  adyacencia;  // vector de vectores
};

Sistema académico compuesto

struct Materia {
    string           nombre;
    int              creditos;
    list<Estudiante*> inscritos;
};

struct Carrera {
    string           nombre;
    list<Estudiante*> estudiantes;
    list<Materia>    materias;
};

Ventajas y desventajas

VentajaDesventaja
Sin duplicación de datosMayor complejidad de implementación
Consistencia automática entre vistasGestión de memoria más delicada
Eficiencia en múltiples tipos de consultaDifícil de depurar
