Clase 33: Preparación del Proyecto Final

Revisión, optimización y documentación antes de la entrega definitiva

Checklist de revisión de código

1. Corrección

  • ☐ Todas las operaciones especificadas están implementadas.
  • ☐ Los casos borde están manejados (estructura vacía, un elemento, datos inválidos).
  • ☐ Las pruebas pasan en todos los escenarios.

2. Eficiencia

  • ☐ La complejidad de cada operación está documentada y justificada.
  • ☐ No hay bucles innecesarios ni búsquedas lineales donde debería haber O(1).
  • ☐ Se eligió la estructura adecuada para cada operación frecuente.

3. Gestión de memoria

  • ☐ Cada new tiene su delete correspondiente.
  • ☐ Los destructores liberan toda la memoria.
  • ☐ No hay referencias a memoria liberada (dangling pointers).

Técnicas de optimización comunes

// Antes: búsqueda O(n) en lista
for (auto& e : lista) if (e.id == id) return e;

// Después: búsqueda O(1) con tabla hash
unordered_map<int, Elemento> tabla;
return tabla[id];  // O(1)

// Antes: verificar existencia O(n)
bool existe = false;
for (auto& e : lista) if (e==val) {existe=true; break;}

// Después: O(1) con unordered_set
unordered_set<int> conjunto;
bool existe = conjunto.count(val) > 0;  // O(1)
