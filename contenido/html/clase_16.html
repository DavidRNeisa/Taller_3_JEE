Clase 16: Árboles de Partición del Espacio

Quadtree, K-d Tree y Octree: búsqueda espacial eficiente

¿Por qué árboles de partición?

Las búsquedas en espacios multidimensionales (¿qué objetos están dentro de este rectángulo? ¿cuál es el vecino más cercano?) son ineficientes con estructuras lineales. Los árboles de partición dividen el espacio recursivamente para acelerar estas consultas.

Quadtree

Divide un plano 2D en cuatro cuadrantes recursivamente. Cada nodo representa una región y tiene hasta 4 hijos.

struct NodoQt {
    double x, y, ancho, alto;  // región
    NodoQt* cuadrantes[4];     // NE, NO, SE, SO
    vector<Punto> puntos;      // puntos en esta región (si es hoja)
};

PSEUDOCÓDIGO insertar(nodo, punto):
  SI nodo es hoja Y puntos.tamaño < MAX:
    puntos.agregar(punto)
  SINO:
    SI no tiene hijos: subdivir en 4 cuadrantes
    insertar en el cuadrante correspondiente

// Búsqueda en rango: O(log n + k) donde k = puntos encontrados

Aplicaciones: compresión de imágenes, detección de colisiones en videojuegos, GIS (sistemas de información geográfica).

Octree

Extensión del Quadtree al espacio 3D: cada nodo se divide en 8 octantes.

Aplicaciones: renderización 3D (culling de objetos fuera del campo de visión), simulaciones de física, nubes de puntos en escáneres 3D.

K-d Tree

Árbol binario que divide el espacio k-dimensional alternando la dimensión de corte en cada nivel.

// Para 2D: nivel par divide por X, nivel impar divide por Y
struct NodoKD {
    double punto[2];   // {x, y}
    NodoKD *izq, *der;
};

NodoKD* construir(vector<Punto>& pts, int profundidad) {
    if (pts.empty()) return nullptr;
    int eje = profundidad % 2;   // 0=X, 1=Y
    // ordenar por el eje actual y tomar el mediano
    sort(pts.begin(), pts.end(), [eje](Punto a, Punto b){ return a[eje]<b[eje]; });
    int mid = pts.size() / 2;
    NodoKD* nodo = new NodoKD{pts[mid]};
    nodo->izq = construir(subvector(pts, 0, mid), profundidad+1);
    nodo->der = construir(subvector(pts, mid+1, pts.size()), profundidad+1);
    return nodo;
}
// Construcción: O(n log n). Búsqueda vecino más cercano: O(log n) promedio
