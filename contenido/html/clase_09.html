Clase 9: Ejercicios Tipo Parcial — Estructuras Lineales

Practica las tres habilidades: analizar, trazar, implementar

Estrategia de examen

  • Analizar complejidad: identifica bucles y recursión, aplica reglas.
  • Trazar ejecución: dibuja el estado de la estructura paso a paso.
  • Implementar: escribe código correcto con manejo de casos borde.

Ejercicio 1: Cola con lista enlazada

struct Nodo { int dato; Nodo* sig=nullptr; };
struct Cola {
    Nodo *frente=nullptr, *fin=nullptr;
    void enqueue(int v) {
        Nodo* n = new Nodo{v};
        if (!fin) { frente=fin=n; return; }
        fin->sig=n; fin=n;
    }
    int dequeue() {
        if (!frente) throw runtime_error("Vacía");
        int v=frente->dato; Nodo* t=frente;
        frente=frente->sig;
        if (!frente) fin=nullptr;
        delete t; return v;
    }
};
// Traza: enqueue(1),enqueue(2),enqueue(3),dequeue()
// Resultado: frente→[2]→[3]←fin

Ejercicio 2: Invertir una pila

stack<int> invertir(stack<int> orig) {
    stack<int> aux;
    while (!orig.empty()) { aux.push(orig.top()); orig.pop(); }
    return aux;
}
// [1,2,3] (3 en cima) → [3,2,1] (1 en cima)

Ejercicio 3: Detectar ciclo — algoritmo de Floyd

bool tieneCiclo(Nodo* cabeza) {
    Nodo *lento=cabeza, *rapido=cabeza;
    while (rapido && rapido->sig) {
        lento  = lento->sig;
        rapido = rapido->sig->sig;
        if (lento==rapido) return true;
    }
    return false;
}
// O(n) tiempo, O(1) espacio extra — sin usar tabla hash

Ejercicio 4: Análisis de complejidad mixta

void misterio(int n) {
    for (int i=1; i<=n; i*=2)    // log₂(n) iteraciones
        for (int j=0; j<n; j++)  // n cada vez
            cout << i+j;
}
// Complejidad: O(n log n)

Ejercicio 5: Encontrar el k-ésimo elemento desde el final

// Sin conocer el tamaño de la lista — dos punteros
Nodo* kDesdeElFinal(Nodo* cabeza, int k) {
    Nodo *adelante=cabeza, *atras=cabeza;
    for (int i=0; i<k; i++) {
        if (!adelante) return nullptr;  // k mayor que el tamaño
        adelante = adelante->sig;
    }
    while (adelante) {
        adelante = adelante->sig;
        atras    = atras->sig;
    }
    return atras;  // apunta al k-ésimo desde el final
}
// Complejidad: O(n) con un solo recorrido
Consejo de examen: siempre dibuja la estructura antes de codificar. Traza el estado de los punteros después de cada operación para evitar referencias colgantes.
