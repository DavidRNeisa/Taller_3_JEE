Clase 3: Tipos Abstractos de Datos (TAD)

Separar el "qué" del "cómo": contrato vs. implementación

Concepto

Un TAD define un conjunto de valores y las operaciones permitidas sobre ellos, sin especificar cómo se implementan internamente. Es el principio de abstracción aplicado a los datos.

Analogía: Una cafetera tiene interfaz clara: añadir agua, añadir café, presionar botón, obtener café. No necesitas saber si usa resistencia eléctrica o gas. El TAD es esa interfaz.

TAD Lista

TAD Lista<T>:
  insertar(pos, elemento)   // O(n)
  eliminar(pos)             // O(n)
  obtener(pos) → T          // O(1) en arreglo, O(n) en lista enlazada
  tamaño() → entero
  estaVacia() → booleano

TAD Pila — LIFO (Last In, First Out)

El último en entrar es el primero en salir. Imagina una pila de platos.

TAD Pila<T>:
  push(elemento)   // agrega arriba
  pop() → T        // quita y devuelve el de arriba
  top() → T        // consulta el de arriba
  estaVacia() → booleano

// En C++:
#include <stack>
stack<int> pila;
pila.push(10); pila.push(20); pila.push(30);
cout << pila.top();  // 30
pila.pop();           // quita 30

Aplicaciones: call stack del SO, deshacer/rehacer, paréntesis balanceados, expresiones postfijas.

TAD Cola — FIFO (First In, First Out)

El primero en entrar es el primero en salir. Imagina una fila del banco.

TAD Cola<T>:
  enqueue(elemento)  // agrega al final
  dequeue() → T      // extrae del frente
  frente() → T
  estaVacia() → booleano

// En C++:
#include <queue>
queue<string> cola;
cola.push("Ana"); cola.push("Luis"); cola.push("Pedro");
cout << cola.front(); // Ana
cola.pop();            // quita Ana

Aplicaciones: gestión de impresión, scheduling de procesos, BFS, buffers de comunicación.

Ventajas de usar TADs

  • Encapsulamiento: el usuario no depende de los detalles internos.
  • Intercambiabilidad: puedes cambiar la implementación sin afectar al código cliente.
  • Reutilización: un TAD bien diseñado sirve en múltiples proyectos.
En C++: los TADs se implementan con clases. La STL provee implementaciones de calidad industrial: vector, list, stack, queue, map, set, etc.
