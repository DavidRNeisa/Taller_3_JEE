Clase 25: Introducción a Grafos

Modelar relaciones complejas: redes, mapas y conexiones

¿Qué es un grafo?

Un grafo G = (V, E) es una estructura formada por un conjunto de vértices (nodos) V y un conjunto de aristas (arcos) E que conectan pares de vértices. Es la estructura más general para modelar relaciones.

Clasificación de grafos

TipoDescripciónEjemplo real
No dirigidoLas aristas no tienen direcciónRed de amistad
Dirigido (digrafo)Las aristas tienen dirección (A→B ≠ B→A)Twitter (seguir)
PonderadoLas aristas tienen pesoMapa de distancias
ConexoHay camino entre cualquier par de nodosInternet
ÁrbolGrafo conexo acíclicoÁrbol genealógico

Representación 1: Matriz de adyacencia

// Para un grafo de n nodos: matriz n×n
// adj[i][j] = 1 si hay arista entre i y j (o el peso si es ponderado)
int adj[4][4] = {
//  0  1  2  3
  { 0, 1, 1, 0 },  // 0 conectado con 1 y 2
  { 1, 0, 0, 1 },  // 1 conectado con 0 y 3
  { 1, 0, 0, 1 },  // 2 conectado con 0 y 3
  { 0, 1, 1, 0 }   // 3 conectado con 1 y 2
};
// Espacio: O(V²). Verificar arista: O(1). Listar vecinos: O(V).
// Ideal para grafos densos (muchas aristas).

Representación 2: Lista de adyacencia

// Para cada nodo, guardar la lista de sus vecinos
vector<vector<int>> adj(4);
adj[0] = {1, 2};
adj[1] = {0, 3};
adj[2] = {0, 3};
adj[3] = {1, 2};

// Con pesos:
vector<vector<pair<int,int>>> adjW(4);
adjW[0].push_back({1, 5});  // arista 0→1 con peso 5
adjW[0].push_back({2, 3});  // arista 0→2 con peso 3

// Espacio: O(V+E). Listar vecinos: O(grado). Verificar arista: O(grado).
// Ideal para grafos dispersos (pocas aristas).

Aplicaciones de grafos

  • GPS/Mapas: ciudades=nodos, carreteras=aristas ponderadas.
  • Redes sociales: personas=nodos, relaciones=aristas.
  • Internet: routers=nodos, conexiones=aristas.
  • Compiladores: dependencias entre módulos.
  • Scheduling: tareas con dependencias entre sí.
