Clase 2: Complejidad Algorítmica y Notación Big-O

Cómo medir y comparar la eficiencia de los algoritmos

¿Por qué medir eficiencia?

Dos algoritmos que resuelven el mismo problema pueden comportarse de forma radicalmente distinta cuando el volumen de datos crece. Medir la eficiencia nos permite predecir el rendimiento y elegir el algoritmo correcto antes de implementarlo.

Ejemplo motivador: Buscar en 1 000 000 de nombres: búsqueda lineal → hasta 1 000 000 pasos. Búsqueda binaria → máximo 20 pasos. La diferencia es astronómica.

Complejidad temporal vs. espacial

  • Temporal: cuántas operaciones realiza en función del tamaño de entrada n.
  • Espacial: cuánta memoria adicional requiere.

Reglas para calcular complejidad

  • Instrucción simple → O(1)
  • Bucle simple de 0 a n → O(n)
  • Bucles anidados independientes → multiplicar: O(n²)
  • Dividir el problema a la mitad cada paso → O(log n)
  • Bloques secuenciales → conservar el mayor

Ejemplos progresivos

// O(1)
int primero(int arr[]) { return arr[0]; }

// O(n)
int suma(int arr[], int n) {
    int s = 0;
    for (int i = 0; i < n; i++) s += arr[i];
    return s;
}

// O(n²)
void pares(int arr[], int n) {
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            cout << arr[i] << "," << arr[j] << "\n";
}

// O(log n)
int busquedaBinaria(int arr[], int n, int x) {
    int izq = 0, der = n-1;
    while (izq <= der) {
        int m = (izq+der)/2;
        if (arr[m]==x) return m;
        else if (arr[m]<x) izq=m+1;
        else der=m-1;
    }
    return -1;
}

Casos de análisis

  • Mejor caso (Ω): escenario más favorable.
  • Caso promedio (Θ): comportamiento esperado con entradas típicas.
  • Peor caso (O): escenario más desfavorable. Es el más usado en diseño.
Error común: Big-O no mide velocidad absoluta, sino crecimiento. Un O(n²) con n=10 puede ser más rápido que un O(n) con constante grande.
