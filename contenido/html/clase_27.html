Clase 27: Caminos de Euler y Hamilton

Dos problemas clásicos con naturaleza computacional muy diferente

Camino y Ciclo de Euler

Un camino de Euler recorre cada arista exactamente una vez. Si regresa al nodo inicial, es un ciclo de Euler. Este problema tiene solución eficiente.

bool tieneCicloEuler(vector<vector<int>>& adj, int V) {
    for (int v = 0; v < V; v++)
        if (adj[v].size() % 2 != 0) return false;
    return esConexo(adj, V);
}

// Algoritmo de Hierholzer para encontrar el ciclo — O(E)
void hierholzer(int inicio, vector<vector<int>>& adj) {
    stack<int> pila;
    vector<int> ciclo;
    pila.push(inicio);
    while (!pila.empty()) {
        int v = pila.top();
        if (adj[v].empty()) { ciclo.push_back(v); pila.pop(); }
        else { pila.push(adj[v].back()); adj[v].pop_back(); }
    }
    for (int v : ciclo) cout << v << " ";
}

Aplicación histórica

El origen: los 7 puentes de Königsberg (1736). Euler demostró que no existía un camino que cruzara cada puente exactamente una vez, fundando la teoría de grafos.

Camino y Ciclo de Hamilton

Un camino de Hamilton visita cada vértice exactamente una vez. Si regresa al inicio, es un ciclo de Hamilton. Este problema es NP-completo — no se conoce algoritmo eficiente general.

// Solución por backtracking — O(n!) en peor caso
vector<int> camino;
vector<bool> visitado;

bool hamilton(int actual, int inicio, int visitados, int V,
              vector<vector<int>>& adj) {
    if (visitados == V) {
        // Verificar si hay arista de vuelta al inicio
        for (int u : adj[actual]) if (u==inicio) return true;
        return false;
    }
    for (int u : adj[actual]) {
        if (!visitado[u]) {
            visitado[u]=true; camino.push_back(u);
            if (hamilton(u, inicio, visitados+1, V, adj)) return true;
            visitado[u]=false; camino.pop_back();
        }
    }
    return false;
}
